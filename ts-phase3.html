Tuần 9 · Page Object Model
🏛️
Typed Page Objects, Components, Locator Ownership
class page objects · private locators · return typed methods
▼
Lý thuyết

TypeScript rất hợp với POM vì method signature và object shape rõ ràng. Bạn có thể trả về page object tiếp theo sau action để test flow đọc như business steps.

TypeScript pages/LoginPage.ts 
export class LoginPage {
  constructor(private page: Page) {}

  async goto() {
    await this.page.goto("/");
  }

  async login(username: string, password: string) {
    await this.page.locator("#user-name").fill(username);
    await this.page.locator("#password").fill(password);
    await this.page.locator("#login-button").click();
  }
}

▼
Lý thuyết

Automation chỉ thực sự hữu ích với team khi lỗi fail được kể lại rõ ràng. Report, screenshot, trace và step log là bằng chứng giúp QA, dev và reviewer cùng hiểu vấn đề.

⚠️
Đừng chỉ assert fail rồi bỏ đó. Hãy đính kèm context: URL hiện tại, screenshot, trace, request/response liên quan hoặc test.step có mô tả business action.

▼
Lý thuyết

Track TS rất mạnh ở mô hình “API setup, UI verify”. Bạn có thể login qua API, lưu storage state rồi tái sử dụng trong hàng chục test UI để chạy nhanh và ổn định hơn.

TypeScript auth.setup.ts 
test("create auth state", async ({ request, page }) => {
  await request.post("/login", {
    data: { username: "standard_user", password: "secret_sauce" },
  });
  await page.context().storageState({ path: ".auth/user.json" });
});
