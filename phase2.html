Python first_script.py 
from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    # Khởi động browser (có thể dùng firefox, webkit)
    browser = p.chromium.launch(headless=False)  # False = hiện cửa sổ
    page    = browser.new_page()

    # Điều hướng
    page.goto("https://www.saucedemo.com")

    # Tương tác
    page.locator("#user-name").fill("standard_user")
    page.locator("#password").fill("secret_sauce")
    page.locator("#login-button").click()

    # Chụp screenshot
    page.screenshot(path="after_login.png")

    print(f"URL hiện tại: {page.url}")
    browser.close()

Locators — Tìm Element
CSS · XPath · getByRole · getByText · getByLabel
▼
Lý thuyết — Thứ tự ưu tiên locator
✅
Playwright khuyến nghị: Ưu tiên dùng role-based locators (getByRole, getByLabel, getByText) thay vì CSS/XPath vì chúng bền hơn khi UI thay đổi.
Python locators.py 
## CSS Selector (quen thuộc với web dev)
page.locator("#user-name")                # id
page.locator(".btn-primary")              # class
page.locator("input[type='submit']")       # attribute
page.locator("form > .login-wrapper input") # descendant

## XPath (mạnh nhưng dễ vỡ)
page.locator("xpath=//button[@class='login-button']")
page.locator("xpath=//span[text()='Add to cart']")

## Playwright-native locators (KHUYẾN NGHỊ DÙNG)
page.get_by_role("button", name="Login")          # theo ARIA role
page.get_by_label("Username")                    # theo <label>
page.get_by_placeholder("Enter email")           # theo placeholder
page.get_by_text("Add to cart")                   # theo text
page.get_by_test_id("checkout-button")           # data-testid attr

## Chaining — tìm bên trong element
product_card = page.locator(".inventory_item").first
product_card.get_by_role("button").click()

## nth, first, last — chọn trong danh sách
page.locator(".product-name").nth(0)    # phần tử đầu tiên
page.locator(".product-name").first    # tương đương nth(0)
page.locator(".product-name").last     # phần tử cuối
Locator Độ bền Dùng khi
getByRole ⭐⭐⭐⭐⭐ Button, link, heading, input có accessible name
getByLabel ⭐⭐⭐⭐⭐ Form field có <label>
getByText ⭐⭐⭐⭐ Text content ổn định
getByTestId ⭐⭐⭐⭐⭐ Dev đã thêm data-testid (chuẩn nhất)
CSS selector ⭐⭐⭐ Khi không có lựa chọn tốt hơn
XPath ⭐⭐ Chỉ dùng khi bắt buộc

Locator Contract — get_by_role, get_by_label, data-testid
selector bền vững · accessibility-first · phối hợp với dev
▼
Kiến thức quan trọng

Biết nhiều selector chưa chắc giúp test bền. Điều quan trọng hơn là thống nhất selector contract: khi nào ưu tiên get_by_role(), khi nào dùng get_by_label(), và khi nào nên yêu cầu dev thêm data-testid.

Python locator_contract.py 
## Ưu tiên 1: role + accessible name
page.get_by_role("button", name="Login")

## Ưu tiên 2: label cho form fields
page.get_by_label("Username")

## Ưu tiên 3: data-testid khi text/UI dễ đổi
page.get_by_test_id("checkout-submit")

## Chỉ fallback về CSS khi không còn cách ổn hơn
page.locator(".login-form button.submit")
✅
Quy tắc team nên nhớ: selector phục vụ automation là một phần của UI contract. Nếu text thay đổi thường xuyên, component sinh động hoặc có nhiều nút giống nhau, hãy xin data-testid thay vì bám vào CSS dễ gãy.
❌
Tránh XPath dài, selector theo vị trí như div:nth-child(3), hoặc locator theo text quá chung như "Save" khi trên trang có nhiều nút giống nhau.

Assertions với expect()
to_be_visible · to_have_text · to_have_url · to_be_enabled
▼
Lý thuyết

Assertion là trái tim của mọi test case — đây là nơi bạn xác nhận rằng ứng dụng đang hoạt động đúng. Playwright cung cấp expect() với auto-retry tích hợp sẵn.

Python assertions.py 
from playwright.sync_api import expect

## Page assertions
expect(page).to_have_url("https://example.com/dashboard")
expect(page).to_have_title("Dashboard")

## Locator assertions — TỰ RETRY cho đến khi pass hoặc timeout
expect(page.locator("h1")).to_be_visible()
expect(page.locator(".error-msg")).to_have_text("Invalid credentials")
expect(page.locator("#submit")).to_be_enabled()
expect(page.locator("#submit")).to_be_disabled()
expect(page.locator("input#email")).to_have_value("[email protected]")
expect(page.locator(".cart-badge")).to_have_text("3")
expect(page.locator("#agree")).to_be_checked()

## NOT — phủ định assertion
expect(page.locator(".error")).not_to_be_visible()

## Custom timeout
expect(page.locator(".spinner")).not_to_be_visible(timeout=10_000)  # 10s

## Soft assertion — ghi nhớ lỗi, không dừng test ngay
from playwright.sync_api import expect
expect.soft(page.locator("h1")).to_have_text("Welcome")
expect.soft(page.locator(".subtitle")).to_be_visible()
page.context.tracing.stop()  # báo lỗi cuối cùng
⚠️
Tránh dùng assert element.is_visible() — cách này không có retry. Luôn dùng expect(locator).to_be_visible() để test ổn định hơn.

Auto-waiting · wait_for_load_state · wait_for_response · page.route()
▼
Lý thuyết — Auto-waiting là gì?

Playwright tự động chờ element trước khi thực hiện action. Khi bạn click(), Playwright chờ element: visible, stable (không đang animate), enabled. Đây là lý do Playwright ổn định hơn Selenium.

Python waits.py 
## TUYỆT ĐỐI TRÁNH — time.sleep() cứng nhắc
import time
time.sleep(3)  # ❌ KHÔNG BAO GIỜ làm thế này!

## ĐÚNG — chờ theo trạng thái trang
page.wait_for_load_state("networkidle")  # chờ hết request
page.wait_for_load_state("domcontentloaded")
page.wait_for_load_state("load")

## Chờ element xuất hiện
page.locator(".result-list").wait_for(state="visible")
page.locator(".spinner").wait_for(state="hidden")  # chờ spinner biến mất

## Chờ navigation sau khi click
with page.expect_navigation():
    page.get_by_role("button", name="Submit").click()

## Mock API — rất hữu ích để test error states!
def mock_error(route):
    route.fulfill(status=500, body='{"error": "Server down"}')

page.route("**/api/products", mock_error)
page.goto("/products")
# Page sẽ nhận response 500, test UI xử lý lỗi

## Intercept response — kiểm tra API call
with page.expect_response("**/api/login") as response_info:
    page.get_by_role("button", name="Login").click()
response = response_info.value
print(response.status)  # 200

Screenshots, Dialogs, iFrames & Multi-tab
screenshot · video · page.on("dialog") · frames · new_page
▼
Lý thuyết

Playwright xử lý được các tình huống phức tạp thường gặp trong ứng dụng thực tế: popup alert/confirm, nội dung trong iframe, và trang mở tab mới. Đây là các kỹ năng bắt buộc khi test ứng dụng production.

Python screenshots_dialogs.py 
## SCREENSHOTS ##
# Chụp toàn bộ trang
page.screenshot(path="full_page.png", full_page=True)

# Chụp một element cụ thể
page.locator(".invoice-table").screenshot(path="invoice.png")

# Bật video recording cho toàn bộ session
context = browser.new_context(record_video_dir="./videos")
page = context.new_page()
# ... chạy test ...
# Video được lưu khi context.close()
context.close()

## DIALOGS (alert / confirm / prompt) ##
# Playwright auto-dismiss dialogs — cần handle thủ công
def handle_dialog(dialog):
    print(f"Dialog type: {dialog.type}, message: {dialog.message}")
    if dialog.type == "confirm":
        dialog.accept()    # nhấn OK
    else:
        dialog.dismiss()   # nhấn Cancel/close

page.on("dialog", handle_dialog)
page.get_by_role("button", name="Delete").click()
# dialog.accept() tự động được gọi

## iFRAMES ##
# Lấy frame theo URL hoặc name
frame = page.frame(url="**/payment-iframe")
frame = page.frame_locator("iframe[name='payment']")

# Tương tác bên trong frame — giống page bình thường
frame.get_by_label("Card number").fill("4111111111111111")
frame.get_by_role("button", name="Pay").click()

## MULTI-TAB ##
# Bắt sự kiện khi trang mở tab mới
with context.expect_page() as new_page_info:
    page.get_by_role("link", name="Open in new tab").click()
new_tab = new_page_info.value
new_tab.wait_for_load_state()
expect(new_tab).to_have_url("https://example.com/detail")
⚠️
Lưu ý dialog: Nếu không đăng ký page.on("dialog"), Playwright sẽ tự dismiss tất cả dialogs. Đăng ký handler trước khi trigger action gây ra dialog.

Lý thuyết — pytest là gì?

pytest là framework testing phổ biến nhất cho Python. Nó cung cấp: auto-discovery test functions, fixtures để setup/cleanup, parametrize để test nhiều data, và reporting đẹp mắt.

Python test_example.py 
import pytest
from playwright.sync_api import Page

def test_login_success(page: Page):
    page.goto("https://saucedemo.com")
    page.locator("#user-name").fill("standard_user")
    page.locator("#password").fill("secret_sauce")
    page.locator("#login-button").click()
    expect(page).to_have_url(re.compile(r".*/inventory.html"))

def test_login_fail(page: Page):
    page.goto("https://saucedemo.com")
    page.locator("#user-name").fill("invalid_user")
    page.locator("#password").fill("wrong_password")
    page.locator("#login-button").click()
    expect(page.locator(".error-message")).to_be_visible()
Python conftest.py 
import pytest
from playwright.sync_api import Playwright, Browser, BrowserContext

@pytest.fixture(scope="session")
def browser(playwright: Playwright) -> Browser:
    browser = playwright.chromium.launch(headless=False)
    yield browser
    browser.close()

@pytest.fixture
def page(browser: Browser) -> Page:
    context = browser.new_context()
    page = context.new_page()
    yield page
    context.close()
Python test_parametrize.py 
@pytest.mark.parametrize("username,password,expected", [
    ("standard_user", "secret_sauce", "success"),
    ("locked_out_user", "secret_sauce", "locked"),
    ("problem_user", "secret_sauce", "problem"),
])
def test_login_scenarios(page: Page, username, password, expected):
    page.goto("https://saucedemo.com")
    page.locator("#user-name").fill(username)
    page.locator("#password").fill(password)
    page.locator("#login-button").click()

    if expected == "success":
        expect(page).to_have_url(re.compile(r".*/inventory.html"))
    elif expected == "locked":
        expect(page.locator(".error")).to_contain_text("locked out")
    elif expected == "problem":
        # Test specific behavior for problem user
        expect(page.locator(".inventory_item")).to_have_count(6)

Lý thuyết — Tổ chức test project

Project testing production cần cấu trúc rõ ràng để dễ maintain và scale. pytest cung cấp nhiều tính năng để tổ chức test hiệu quả.

pytest.ini 
[tool:pytest]
addopts =
    --browser chromium
    --headed
    --slowmo 500
    --video retain-on-failure
    --screenshot only-on-failure
    --html=reports/report.html
    --self-contained-html
markers =
    smoke: quick smoke tests
    regression: full regression suite
    slow: tests that take >30 seconds
    api: API-only tests
    ui: UI-only tests
