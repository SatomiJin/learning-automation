Tuần 13 · Angular DOM Reality
🧱
Angular Source: class và id có thể dynamic
_ngcontent · ng-star-inserted · generated ids · selector traps
▼
Kiến thức thực chiến

Trong Angular app thật, nhiều id, class hoặc attribute như _ngcontent-*, ng-star-inserted, cdk-* được generate động. Nếu test bám vào các giá trị này, test sẽ flaky hoặc vỡ ngay khi UI refactor nhẹ.

Python selector_strategy.py 
# ❌ Đừng bám vào generated class/id
page.locator(".ng-star-inserted .ant-select-selection-item")
page.locator("#mat-select-12")

# ✅ Ưu tiên container ổn định + accessible selector
user_form = page.locator("[data-testid='user-form']")
user_form.get_by_label("Department").click()
page.get_by_role("option", name="Finance").click()
⚠️
Nguyên tắc vàng: đừng hỏi “selector nào chạy được”, hãy hỏi “selector nào sẽ còn sống sau 3 sprint nữa”.

Tuần 14 · AntD & Material Controls
🧰
Select, Search Select, Date Picker, Dialog
AntD overlay · mat-select · autocomplete · custom input behavior
▼
Kiến thức thực chiến

Ant Design và Angular Material thường không dùng <select> HTML thuần. Chúng render dropdown qua overlay hoặc portal, nên test phải click đúng control, chờ list xuất hiện rồi chọn option từ DOM mới được render.

Python antd_select.py 
# Ant Design searchable select
page.get_by_label("Assignee").click()
page.locator(".ant-select-selection-search input").fill("linh")
page.get_by_role("option", name="Linh Nguyen").click()

# Angular Material select
page.get_by_label("Status").click()
page.get_by_role("option", name="Approved").click()
💡
Nếu click rồi không thấy option, hãy nghĩ tới overlay render ngoài component gốc chứ đừng vội tăng sleep.

Tuần 15 · Table, Overlay, Virtual Scroll
📊
Data Table, Pagination, Sticky Header, Virtual List
ant-table · mat-table · business key assertions · lazy rendering
▼
Kiến thức thực chiến

Enterprise app rất hay có table lớn, filter nhiều tầng, sticky header và virtual scroll. Khi đó, item bạn cần có thể chưa render trong DOM dù “về mặt business” nó đã có trên màn hình danh sách.

Python table_waits.py 
# Chờ API trả dữ liệu xong rồi mới assert table
with page.expect_response("**/api/orders*") as response_info:
    page.get_by_role("button", name="Search").click()

response = response_info.value
assert response.ok

table = page.locator("[data-testid='orders-table']")
expect(table).to_contain_text("ORD-10025")
❌
Đừng assert row theo nth(3) nếu table có sort, filter, pagination hoặc virtual scroll. Hãy assert theo nội dung business key như mã đơn, email hoặc trạng thái.

Tuần 16 · Debugging & Dev Contract
🤝
Đọc Source, Yêu Cầu Test Hooks, Chống Flaky Dài Hạn
component tree · data-testid request · loading states · source reading
▼
Kiến thức thực chiến

QC automation làm tốt không chỉ là “viết selector chạy được”, mà còn biết khi nào nên đọc source component, khi nào nên push back dev để thêm data-testid, aria-label hoặc tách loading state rõ ràng hơn.

✅
Checklist khi test enterprise UI khó: xác định business key, hiểu control render ở đâu, biết API nào nuôi data, xem loading state, và thống nhất test hooks với dev nếu DOM không semantic.
⚠️
Nếu một control buộc bạn phải bám vào class generated hoặc XPath quá dài, rất có thể đây là dấu hiệu nên đề xuất thay đổi source thay vì cố viết test “mẹo”.
