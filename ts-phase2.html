🎯
Locator Contract: getByRole, getByLabel, data-testid
accessibility-first · stable selectors · phối hợp với dev
▼
Kiến thức quan trọng

Phần lớn test flaky đến từ selector không ổn định. Khi học Playwright, điều quan trọng không phải là “biết nhiều selector”, mà là biết ưu tiên selector nào trước và khi nào cần yêu cầu dev thêm data-testid.

TypeScript locator-priority.ts 
// Ưu tiên 1: accessible role/name
page.getByRole("button", { name: "Login" });

// Ưu tiên 2: label / placeholder nếu là form field
page.getByLabel("Username");

// Ưu tiên 3: data-testid khi UI text hay thay đổi
page.getByTestId("checkout-submit");

// Chỉ fallback về CSS khi không còn lựa chọn tốt hơn
page.locator(".login-form button.submit");
✅
Quy tắc team nên thống nhất: selector phục vụ automation là một phần contract của UI. Nếu text thay đổi thường xuyên hoặc component quá động, hãy xin data-testid thay vì bám vào CSS dễ vỡ.
❌
Tránh XPath dài, selector theo vị trí như div:nth-child(3), hoặc text quá chung chung như "Save" khi trên trang có nhiều nút cùng tên.

Tuần 6 · Actions & Assertions
🖱️
Actions, Auto-waiting, Typed Assertions
click · fill · select · expect() · readable test flow
▼
Lý thuyết

Điểm mạnh của Playwright là auto-waiting. Khi viết test TypeScript, bạn thường kết hợp action rõ ràng với assertion ngắn và typed để flow vừa chắc vừa dễ review.

TypeScript tests/cart.spec.ts 
test("add one item to cart", async ({ page }) => {
  await page.goto("https://www.saucedemo.com");
  await page.locator("#user-name").fill("standard_user");
  await page.locator("#password").fill("secret_sauce");
  await page.locator("#login-button").click();
  await page.getByRole("button", { name: "Add to cart" }).first().click();
  await expect(page.locator(".shopping_cart_badge")).toHaveText("1");
});

Tuần 7 · Waits & Network
🌐
Route Mocking, Network Assertions, Debug Tools
page.route · waitForResponse · trace viewer
▼
Lý thuyết

Với ứng dụng hiện đại, UI test tốt thường phải hiểu network. Mock đúng response sẽ giúp bạn cover nhiều case lỗi mà không phụ thuộc backend.

TypeScript tests/network.spec.ts 
test("mock products api", async ({ page }) => {
  await page.route("**/api/products", async (route) => {
    await route.fulfill({
      status: 200,
      body: JSON.stringify([{ id: 1, name: "Mock Product" }]),
    });
  });
});
